# Сайт банкетного зала «Лазурный» (Чита)

Многостраничный статический сайт (HTML/CSS/JS без сборщика) для банкетного зала
«Лазурный», ул. Энергетиков, 5, Чита. Собран из исходного лендинга по ТЗ
`promt-sayt-lazurny.md` с наследованием дизайн-системы.

## Структура

```
/
├── index.html         Главная (герой, о зале, мини-конфигуратор, отзывы, форма)
├── hall.html          Зал и рассадка (конфигуратор + тех. характеристики)
├── banquets.html      Банкеты и мероприятия (свадьбы/юбилеи/корпоративы/выпускные)
├── menu.html          Меню (три кухни, шаблон под наполнение)
├── gallery.html       Галерея с лайтбоксом
├── reviews.html       Отзывы (пересказ реальных + ссылки на карточки)
├── contacts.html      Контакты, карта, форма заявки
├── robots.txt · sitemap.xml
├── assets/css/styles.css   Общая дизайн-система (палитра/шрифты/компоненты)
├── assets/js/main.js       Общий JS (меню, конфигуратор, лайтбокс, форма)
├── content/                Исходный контент-пакет (reviews.json, menu-template.json, доки)
└── photos/                 Папки под реальные фото + SHOT-LIST.md
```

## Что нужно заменить перед публикацией

1. **Домен.** Везде стоит placeholder `https://lazurny-chita.ru` — в `<link rel="canonical">`,
   Open Graph, `sitemap.xml`, `robots.txt`. Замените на реальный домен (поиск-замена по проекту).
2. **Доставка заявок.** По умолчанию форма отправляет заявку в **WhatsApp** кафе
   (`wa.me/79243889938`) — работает сразу, без бэкенда. Чтобы заявки шли на почту,
   впишите Formspree ID в `FORMSPREE_ENDPOINT` в `assets/js/main.js` — тогда форма
   отправит POST, а WhatsApp останется запасным вариантом.
3. **Меню.** В `menu.html` — плейсхолдеры `[название блюда]` / `[цена]`. Заполните
   реальными позициями, когда клиент пришлёт прайс/фото бумажного меню (см.
   `content/menu-template.json`).
4. **Фото.** Разложите реальные снимки по папкам `photos/*` с именами из
   `photos/SHOT-LIST.md`, затем замените плитки-плейсхолдеры `<figure class="gN">`
   в `gallery.html` на `<img src="/photos/..." alt="..." loading="lazy">`.
5. **Отзывы.** На `reviews.html` кнопки ведут на поиск карточки в 2ГИС/Google.
   Замените на прямые ссылки карточки кафе, когда они будут. Про живой виджет
   отзывов — `content/reviews-widget-instructions.md`.
6. **Метраж зала.** В `hall.html` стоят пометки `[уточнить у клиента]` (площадь).

## Честность контента

Специально **не выдуманы**: меню, цены, точный метраж, дословные цитаты отзывов и
стоковые фото. Где данных нет — стоят плейсхолдеры или оформленные CSS-заглушки.
Подробнее — в `content/README`-файлах и `photos/SHOT-LIST.md`.

## Локальный запуск

Статические файлы — откройте `index.html` в браузере или поднимите локальный сервер:

```bash
python3 -m http.server 8000
# затем http://localhost:8000
```

Используйте сервер (а не `file://`), чтобы корректно работали абсолютные пути
`/assets/...` и переходы между страницами.
