# Шот-лист: какие фото нужны сайту и куда их класть

Реальных фотографий зала «Лазурный» для использования на сайте у меня нет — фото на
2ГИС/Zoon/Google принадлежат тем, кто их туда загрузил (гости или сам персонал кафе),
и просто скачать их для чужого использования на сайте нельзя. Ниже — что нужно
сфотографировать (можно на телефон, не обязательно нанимать фотографа для первой версии
сайта), с точными именами файлов, чтобы Claude Code сразу разложил их по местам без
дополнительных вопросов.

## /photos/hero/ — 1 фото, для фона главного экрана

- `hero-main.jpg` — общий план зала с накрытыми столами (желательно вечером, при свете,
  горизонтальная ориентация, широкий кадр). Заменяет CSS-анимацию свечи в текущем
  лендинге — можно использовать оба варианта, наложив фото под сцену.

## /photos/hall/ — 4-6 фото, для раздела «Зал и рассадка»

- `hall-wide-01.jpg`, `hall-wide-02.jpg` — общие планы зала с разных углов
- `hall-stage.jpg` — сцена и танцпол
- `hall-parking.jpg` — вид парковки у входа (аргумент «бесплатная парковка» из лендинга)
- `hall-entrance.jpg` — вход в кафе снаружи

## /photos/food/ — 6-9 фото, для раздела «Меню»

- `food-01.jpg` … `food-09.jpg` — блюда по 2-3 на каждую кухню (европейская/русская/
  китайская). Снимать сверху при дневном свете, без вспышки, на нейтральном фоне стола.

## /photos/events/ — 4-6 фото, для галереи и раздела «Банкеты»

- `event-wedding-01.jpg` — свадебная рассадка/декор (если разрешат гости прошедшей свадьбы)
- `event-birthday-01.jpg` — оформление на день рождения/юбилей
- `event-corporate-01.jpg` — корпоратив
- `event-table-setting.jpg` — крупный план сервировки стола

## /photos/exterior/ — 1-2 фото

- `exterior-facade.jpg` — фасад здания с вывеской, для карточки в Google/2ГИС и футера сайта

## Технические требования к файлам

- Формат: JPG или PNG, при экспорте для сайта Claude Code сам сожмёт/сконвертирует в WebP
- Ориентация: горизонтальная для hero и hall, любая для food
- Минимальная сторона: 1600 px (чтобы не размывалось на больших экранах)
- Называть файлы латиницей строго как в списке выше — тогда их можно просто скопировать
  в папку, и верстка подхватит их по существующим именам без правки кода

## Если фото ещё нет на момент сборки сайта

Оставить в вёрстке текущие цветные CSS-плейсхолдеры из лендинга (`.g1`–`.g5` в галерее,
анимация свечи в hero) — они уже оформлены прилично и не выглядят как заглушка-ошибка.
Как только реальные файлы появятся в этих папках — заменить `<div class="g1">...</div>`
на `<img src="/photos/hall/hall-wide-01.jpg" alt="Главный зал кафе Лазурный" loading="lazy">`
и так далее по остальным блокам.
